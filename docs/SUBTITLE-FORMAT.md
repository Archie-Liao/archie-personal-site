# 字幕稿排版规则（唯一规范）

> **AI 处理 `design-demos/视频日记字幕稿/` → `public/subtitles/` 时必须读本文件。**  
> 用户不必每次重复口述；若规则需改，只改本文 + `scripts/format-subtitles.mjs`。

## 来源

- **源文件**：`design-demos/视频日记字幕稿/{期数}：{标题}.txt`（语音导出，一行一口气）
- **发布文件**：`public/subtitles/ep{NNN}.txt`（轻整理后供 `SubtitleViewer` 展示）

## 可以改

| 项 | 做法 |
|----|------|
| **标点** | 按口语节奏补 `，。！？`；列表用 `，` 连接 |
| **断句** | 源稿换行常为 mid-phrase，**直接拼接**后再补标点 |
| **分段** | 见下节「分段规则」— **禁止**整篇挤在 1～2 个 `<p>` |
| **重复口语** | 保留（如「感谢感谢」），只加标点 |

## 不可以改

| 项 | 说明 |
|----|------|
| **用字用词** | 以源 TXT 为准 |
| **错字** | 仅当 **≥80% 确定** 为识别/导出错字时才改 |
| **时间轴** | 不恢复 |

## 分段规则（落地标准）

输出 TXT：**一行 = 网站上的一个 `<p>`**（段间空一行）。

1. **句号分段（默认）**  
   每个完整句子单独成段——以 `。` `！` `？` 收束处断开。  
   例：ep099 正文约 8～12 段，而不是 1 大段。

2. **主题段（强制另起段）**  
   - 日期开场（`今天是…号/日`）  
   - `感谢…` 致谢  
   - `终身学习` / `与君共勉` / `明天见` 收尾  

3. **超长句（次要）**  
   单句 **>120 字** 且含 `，；` 时，可在从句边界再拆为两段（仍保留完整语义，不拦腰截断）。

4. **禁止**  
   - 仅按源文件「大块」合并成 1 段  
   - 仅按固定字数机械切分（必须在 `。！？` 或从句边界）

## 句内标点（脚本启发式）

1. 相邻导出行默认用 `，` 连接；换意群用 `。`  
2. 导出断词：`看来当`+`至少` → `看来，至少`（见脚本 `cleanup`）  
3. 末句无句号则补 `。`

## 展示（React）

- `SubtitleViewer`：只读 TXT；**空行 → 新 `<p>`**  
- 不在浏览器里二次断句

## 工作流

```powershell
node scripts/format-subtitles.mjs
npm run dev   # → /post/ep099
```
