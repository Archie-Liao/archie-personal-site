## 这份文档解决什么问题

以前信息散在 `guidelines/`（旧计划）、`AGENTS.md`（给 AI）、`posts.ts`（数据）、`design-demos/`（探索稿）——**没有一份「网站现在长什么样」的单一说明书**。

| 文档 | 用途 | 会不会过时 |
|------|------|------------|
| **本文件 `SITE-MANUAL.md`** | **网站现状 + 线框 + 功能**（给人看） | AI 改页面后同步 |
| [`STATUS.md`](STATUS.md) | 进度、下一步、阻塞 | 收工更新 |
| [`guidelines/个人网站改版方案.md`](../guidelines/个人网站改版方案.md) | 产品需求（稳定，少改） | 仅重大需求变更 |
| [`AGENTS.md`](../AGENTS.md) | AI 协作规则 | 规则变更时 |
| `src/app/data/posts.ts` | **当前**内容数据（mock） | 加条目时 |

**AI 读站顺序**：`STATUS.md` → **本文件** → 再读具体 `src/` 文件。  
**你读站顺序**：**本文件** → 需要细节时打开对应页面预览 `npm run dev`。

**图例**：`[已实现]` = 代码里已有 · `[规划]` = 已定方向未写代码
