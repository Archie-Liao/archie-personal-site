## Must / Never（你的底线，可改）

> **维护规则（硬）**：本节 **只增添、不删改**已有 bullet。新底线 → 追加新行；措辞微调 → 追加一行并标注「取代」旧句，**勿** silent 覆盖。AI 协助填试吃时**不得**整段替换本节。

- **Must**：暖奶油纸底；英文/数字 folio 大排印（benchmark 亮点）；信息分明、可读；有复古文气但不性冷淡

- **Never**：Logo 字体现在不纠结（搁置）

- **Never**：**照猫画虎只换色调**
- **Never**：**压抑黑底长文块**（D2 blackboard）

## benchmark.html（出版物流 · 推荐基准）

- **整站印象（1 句）**：整体良好，清新明朗；淡黄 + 红赭复古配色；Playfair 类英文数字与中文协调——**气质最接近想要的 production 方向**（目前仅首页）。
- **喜欢**：
  - **Hero `Archie.` + `.` dot**：display 英文与点缀小点结合；字体选择好
  - **版画 `figcaption.cap`**：装饰图下说明文字（拉丁名 / PL. 编号）— 空处有了恰当信息，精致感
- **不要**：（本条未提整站级拒绝项）
- **可偷元件**（≥3）：
  - **band 时钟 `#bj-time` + Day 大数字**：时间/数字字体错落有致（D3 stat-band 应对标此精致度）
  - **卷首语 `note-inner`**：首字放大异色；整体卷首语区块
  - **精选卡 `big-folio`**：半透明灰 folio 数字 / ★ — 信息多的卡片有张有弛、多一层深度
- **动效**：几乎无；可接受
- **备注**：前期只做了首页；**整站风格偏好高**，混搭时 Color/Type/Texture 优先从本版偷

## press.html（D1 · Stripe Press）

- **整站印象（1 句）**：平淡温馨，略偏「女生站」气质；不好不坏，不当整站唯一方向。
- **喜欢**：
  - **Warhol 网点**纹理 — 淡、不抢眼，无意识增加信息量
  - **band-inner** 整栏（北京时间 + Day + 迷你月历）风格不错
- **不要**：
  - **精选卡 `is-pick` 大面积变红**区分 — 视觉误导；若区分应用 **字体色 / 装饰**，勿大面积改卡片底色
- **可偷元件**（≥3）：
  - **hero-grid** Hero 标题后 **随机色块** kicker
  - **blackboard** 边框气质可偷 — 内文排版须加强主次（现浏览不流畅）
  - **字幕块**平实清晰排列 — 长文区 OK；**总结 / 思维导图**因字少更要精致排版
- **动效**：暂无；**后续 demo 须多加动效**才能试吃判断
- **备注**：
  - **mindmap** 太呆板 → v1 要更灵动：点击动效 / 可拖词条 / 排布方式优化
  - blackboard、mindmap 与 D3 同属「少字块要设计过」一类

## monocle.html（D2 · Monocle 杂志）

- **整站印象（1 句）**：风格强烈，红黑像热点新闻/杂志站；**可借鉴思路，不当整站肤色**（勿照猫画虎换色调）。
- **喜欢**：
  - **ticker** 滚动横幅 — 创意好；个人站可能怪（红太跳），待定小面积或降饱和
  - **stat-band** 整栏（北京时间 + Day + 月历）— 感受良好
  - **logo `.issue` 102** — 期数作 Logo 不错；方形色块 + 数字字体仍偏普通，要更精致 display
- **不要**：
  - **blackboard 黑底文本框** — 压抑，不宜长时间阅读
- **可偷元件**（≥3）：
  - **tl-row** 时间线选中动效
  - **列表双栏对比**（左黑列表 + 右淡黄预览）— **氛围思路**可借，色调改 benchmark 暖纸 + 赤陶 accent
  - stat-band 结构（字体对齐 benchmark folio）
  - （慎用）ticker — 仅当弱化红色后
- **动效**：tl-row hover/选中不错；**D4/D5 须多做动效样例**供试吃选择
- **备注**：杂志感强 ≠ production 整站；元件级借鉴

## midcentury.html（D3 · 中古海报）

- **整站印象（1 句）**：有独特风格、色彩和谐有个性；但对个人站来说**整体偏浮夸轻浮**，不当整站方向，当元件库。
- **喜欢**：
  - 三色搭配和谐、有记忆点（production 可降饱和 / 小面积用）
  - **poster-frame**：圆形版画 + **三角 accent**（像放大镜）— 视觉亮点
  - **footer** 整体排版（学校名 / 双语文案 / SITE / ELSEWHERE 分区）
- **不要**：
  - **整站三色块铺法** — 对个人站太花
  - **stat-band** 两格（北京时间 / Day 102）：排版偏朴素，数字与中文 **display 装饰不足**，缺 folio 级排印感（benchmark 英文数字排印才是 Must）
  - **粗黑边框 / 分割线**：`detail-hero` 底边、`mindmap` 外框等 **3–4px 纯黑描边** — 太莽；思维导图可偷节点逻辑，**不要**这套边框气质
- **可偷元件**（≥3）：
  - **poster-left Hero**：标题后 **期数 / VOL / 色块 kicker**（「把日记当海报翻阅」）
  - **brand.badge**：左上角用 **期数数字** 作 Logo 元素 — **字体须装饰性、精致**，拒绝黑体数字（low）
  - **list-sidebar**：列表页 **侧栏标签 + 年月索引** 筛选（比顶栏 pill 更杂志感）
  - **index-row hover**：首页时间线选中 / hover 动效（芥末条滑入）
  - （次要）footer 分区结构；poster-frame 圆形+三角（Hero 或关于页）
- **动效**：保留 **index-row** 列表项 hover / 选中反馈；全页 Bass 块面滑入可 subtle 或 Tweaks 默认关
- **备注**：
  - v0；D3 **List 双栏**：左 `list-sidebar` 已有筛选，右 **`list-main` 预览区偏空** — 见下方 IA 评估（应加什么：图表/lead 图/相关链接/文内双链预览？）
  - stat-band 若偷结构，须换 **Playfair 或同类 display 数字** + 装饰线，对齐 Must

## 合成决策（D1–D5 齐了之后填）

> **维护规则（硬）**：表格单元格 **只追加**新来源/新元件（用 `；` 连接）；**禁止**整格覆盖删掉旧表述。定稿前为草稿，仍遵守只增不改。

| 维度 | 从哪版偷 | 具体元件 / 约束 |
|------|----------|-----------------|
| Color | benchmark（淡黄+红赭）+ D1 网点纹理 | 整页色块勿学 D3/D1 pick 大面积 |
| Type | benchmark（Playfair folio + Archie. dot） | Day/时钟 band 对齐 benchmark |
| Layout | benchmark 卷首语 + 精选 folio | D3 期数 kicker；D1 hero 色块 |

| Texture | D1 Warhol 网点 + benchmark 纸感 | 淡、不抢眼 |
| UX | benchmark cap 图注；D1 字幕平排；D2 双栏列表思路 | 少字块（总结/导图）要精致排版；不要 D2 黑底 blackboard |
