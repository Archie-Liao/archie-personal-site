## 二、技术形态（一口饭版）

```text
访客浏览器（前端 React）
  → 发「用户说了啥」
  → 云函数 / 简易后端（后厨，藏 DeepSeek Key）
  → DeepSeek API
  → 答案回到聊天框
```


| 层   | 放什么              | 本仓库大概路径（落地时）                               |
| --- | ---------------- | ------------------------------------------ |
| UI  | 头像、气泡、输入框、开关     | `src/app/components/Assistant*` · 样式独立 css |
| 配置  | 开关文案、模型名（无密钥）    | `site.config.ts` 一段                        |
| 代理  | 调 DeepSeek、限流、日志 | **CloudBase 云函数**（或等价）；密钥只在云端环境变量          |
| 文档  | 本文件 + STATUS     | 勿散落多份互相打架                                  |

### 3.1 云函数：谁申请、谁写码？（和静态部署像不像）

| | **你（Archie）** | **AI** |
|--|------------------|--------|
| 账号 / 实名 / 开通能力 | ✅ 腾讯云里点开通「云函数」或 CloudBase 云函数 | ❌ 我登不了你的控制台 |
| DeepSeek 注册、充值、复制 API Key | ✅ 你办；**只贴进云端环境变量**，别发到聊天/Git | ❌ |
| 函数代码、前端对接、检查清单 | — | ✅ 写好代码 + 逐步点哪 |

**和之前静态站部署的关系**：还是**同一套腾讯云 / CloudBase 账号**，多开一个能力，**不是**另开一家云。  
静态托管 = 摆橱窗（页面）；云函数 = 后厨窗口（代喊 DeepSeek）。  
流程仍像上次：我准备「填什么、点哪」→ 你在控制台点确认 → 密钥只有你自己粘贴。
