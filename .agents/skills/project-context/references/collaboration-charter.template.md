## 一、复杂任务

### 1.1 触发（命中任一）

- 架构 / 数据流 / 多系统
- 新增路由或改信息架构
- 不可逆或高返工
- 需求跨多条线
- 维护者说：**探讨、评估、规划、矩阵、盘点、方案、先写计划**

### 1.2 四步

| 步 | 做什么 | 自检 |
|----|--------|------|
| **1 复述需求** | 目标、约束、**非目标** | 首条含 `## 复述需求` |
| **2 信息清单** | 5～10 条「没有则一定做错」 | 按本任务类型列 |
| **3 完备度声明** | 已满足 / 仍缺 / 缺了会怎样 | 缺关键项 → **只规划不写码** |
| **4 执行** | 小步试点 | 不偷偷扩大范围 |

硬触发细则 → `docs/COMPLEX-TASK-GATE.md`（由 gotchas 索引）

### 1.3 多源任务执行计划（§1.8）

**规定什么**：任务同时依赖 **≥2 个权威文件** 时，**动手前**必须先外化优先级表。禁止在脑内合并后直写代码。

**使用方式**：

1. 输出 **`## 执行计划`**，必含 **参考来源优先级表**：完整路径 · 本任务从该源取什么 · 冲突时服从谁
2. 可选：执行步骤、硬天花板、交付物
3. **默认**在当次回复；跨会话可落盘 `docs/sessions/`
4. **用户确认后再动手**；已开工发现漏读 → **停止** → 补计划 → 继续
5. **禁止**：只读子文件（如单个 recipe / 单章）而声称已执行全文 Skill 或宪章

**计划最小模板**：

```markdown
## 执行计划 · [任务]
### 参考来源与优先级
| 优先级 | 参考文件 | 本任务取什么 | 冲突时 |
| 1 | … | … | 以本行为准 |
### 需确认后再动手
- [ ] 优先级表无误
```

**本项目细则**（路径、案例、任务起点表）→ `docs/MULTI-SOURCE-EXEC-PLAN.md`
