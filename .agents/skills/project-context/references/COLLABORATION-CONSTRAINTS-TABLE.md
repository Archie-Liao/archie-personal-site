# AI 协作规则约束表（跨项目可复用）

> **用途**：复制到任意项目，作为维护者 × AI 的**硬约束速查表**。  
> **来源**：本仓库 [`docs/AI-COLLABORATION-CHARTER.md`](../../../docs/AI-COLLABORATION-CHARTER.md) 的通用部分（已去掉建站/女娲等项目专有条款）。  
> **完整落盘**：若要从零搭文档体系，复制整个 [`.agents/skills/project-context/`](../) 并跑 `scaffold.mjs`。  
> **自检**：去掉本仓库任何路径后，下表是否仍成立？否 → 那条是项目细则，不要写进本表。

**建议放入新项目**：`docs/AI-COLLABORATION-CONSTRAINTS.md`（或 `docs/AI-COLLABORATION-CHARTER.md`），并在 `AGENTS.md` + `.cursor/rules` 各加一行索引。

---

## 0 · 文档怎么分工（五层）

| 层 | 放什么 | 典型文件 | 进 AI 上下文 |
|----|--------|----------|--------------|
| **硬约束** | 短、稳定、每会话必守 | `.cursor/rules/*.mdc`（`alwaysApply`） | 每会话自动 |
| **入口** | 索引 + 短摘要 | `AGENTS.md` | 每会话自动（若作工作区规则） |
| **协作宪章/约束表** | 通用方法论细则 | 本文件或 `AI-COLLABORATION-CHARTER.md` | **不**自动全文 → 复杂任务/改规则时 Read |
| **进度** | 下一步、最近决策 | `docs/STATUS.md` | 新任务先读 |
| **专题 / 域** | 路径、案例、产品需求 | `docs/*`、`guidelines/` | 点名或 `@` |

**禁止**：指望「说过一次 AI 永远记得」——必须落盘；禁止只改代码不改权威 doc。

## 1 · 复杂任务（信息够不够）

### 1.1 何时启用（命中任一）

| 触发 | 示例 |
|------|------|
| 架构 / 数据流 / 多系统 | 前后端、库与应用、多仓库 |
| 改信息架构 | 新路由、新模块边界 |
| 不可逆或高返工 | 内容格式、目录大迁移 |
| 维护者说含糊词 | 「探讨」「你定标准」「不确定何时做」 |
| 跨多条线 | 视觉 + 数据 + 流程同时动 |
| **≥2 个权威源** | Skill + docs + 代码规范冲突可能 | → 同时启用 **§3 多源计划** |

### 1.2 底层原则

| # | 约束 | 含义 |
|---|------|------|
| 1 | **先外化，再行动** | 复述 + 清单 = 可检查假设 |
| 2 | **清单随任务变** | 不套用固定万能表 |
| 3 | **完备度是责任声明** | 已满足 / 仍缺 / 缺了会怎样；缺关键项 → **只规划不写码** |
| 4 | **默认小步试点** | 先小范围再铺开 |

### 1.3 操作四步

| 步 | 必须做 | 硬约束 |
|----|--------|--------|
| 1 复述需求 | 目标、约束、**非目标** | 复杂任务首条回复含 `## 复述需求` |
| 2 信息清单 | 5～10 条「没有则一定做错」 | 按本任务列，勿抄别的 Gate |
| 3 完备度声明 | 已满足 / 仍缺 / 后果 | 缺关键项 → **停止写码** |
| 4 执行或归档 | 小步试点，或只规划 | 禁止偷偷扩大范围 |

### 1.4 收工自检

- [ ] 是否命中触发却直接写码？
- [ ] 是否做了复述 + 完备度？
- [ ] 多源是否先出执行计划？
- [ ] 决策是否写入 STATUS「最近决策」？
- [ ] **是否删短/缩写了本约束表或宪章核心节？**（禁止）

## 2 · 复杂判断（决策可复核）

| # | 约束 | 含义 |
|---|------|------|
| 1 | **判断要留证据链** | 能回答「为何 A 不 B」 |
| 2 | **「不做 / 延后」合法** | 与「现在做」同等比较 |
| 3 | **量化对抗直觉** | 3～6 维度 + 权重或 1–5 分 |
| 4 | **推荐与假设分离** | 前提不成立时推荐如何变 |
| 5 | **时机单独判断** | 「对不对」≠「现在做对不对」 |

| 步 | 必须做 |
|----|--------|
| 1 | ≥2 选项，含「不做 / 延后」 |
| 2 | 可量化标准表 |
| 3 | 推荐 + 未验证假设 |
| 4 | 时机：现在 / 延后 / 某条件后 + 一句理由 |

## 3 · 多源任务执行计划

**何时**：同时依赖 ≥2 个权威文件，且可能冲突。

| 约束 | 内容 |
|------|------|
| **必须** | 动手前输出 `## 执行计划` + **参考来源优先级表** |
| **表列** | 优先级 · 完整路径 · 本任务取什么 · 冲突时服从谁 |
| **确认** | 维护者确认后再动手；漏读 → 停 → 补计划 → 再继续 |
| **禁止** | 脑内合并直写码；只读子文件却声称已读 Skill/宪章全文 |

**最小模板**：

```markdown
## 执行计划 · [任务]
### 参考来源与优先级
| 优先级 | 参考文件 | 本任务取什么 | 冲突时 |
| 1 | … | … | 以本行为准 |
### 需确认后再动手
- [ ] 优先级表无误
```

## 4 · 对话归档（若项目需要语料）

| 项 | 约束 |
|----|------|
| 触发词 | 「整理对话」「归档语料」「保存这次讨论」→ **立即**写文件 |
| 路径 | `knowledge/raw/dialogues/YYYY-MM-DD-主题.md`（可按项目改） |
| 读序 | **最新轮次在上**（#00N → #001） |
| 轮次 | 维护者第 N 次发送 = `#00N`，从 1 递增；**不用** transcript 行号 |
| 维护者原文 | **逐字**；禁止摘要/缩写/「…」代替；优先从 agent transcript 粘贴 |
| Assistant | 仅一行摘要或省略 |
| 时间 | 实时归档到秒（时区在 STATUS/文首约定） |

## 5 · 新需求覆盖旧需求

| 约束 | 内容 |
|------|------|
| **默认不删** | 旧 md/方案用 `status` + 修订记录标记，或移 `_archive/` |
| **权威源分散** | 现状说明 / STATUS / 专题 doc — **不是**堆一个万能文件 |
| **对话中新需求** | **即时同步**权威 doc（与改码同步或先于改码）；禁止只改代码等收工 |
| **覆盖清单** | ① 更新权威源 ② STATUS「最近决策」一行 ③ 旧文件标状态 ④ 修订记录置顶 |

**冲突优先级（通用骨架 · 项目可改名）**：

| 高 → 低 |
|---------|
| 代码/`src` 真源 → 现状说明书 → STATUS → 专题规范 → 早期需求稿 |

## 6 · 规则写在哪（新增规则顺序）

| 写什么 | 写哪 |
|--------|------|
| 每会话 ≤15 条硬约束 | `.cursor/rules/*.mdc` + `alwaysApply` |
| **全局可复用**协作规则 | 宪章 / **本约束表** |
| 本项目路径、案例、起点表 | `docs/` 专题 |
| 进度 / 决策 | `STATUS.md` |
| 一次性长讨论 | `docs/sessions/YYYY-MM-DD.md` |

**顺序**：① 通用句 → 宪章/本表 → ② 项目细则 → 专题 → ③ **更新 AGENTS 索引 + gotchas 一行** → ④ STATUS 一行。

**禁止**：只改宪章不同步索引；禁止为「精简」删掉复杂任务/判断全文。

## 7 · 读档优先级（新会话）

| 优先级 | 读什么 |
|--------|--------|
| P0 | `AGENTS.md` + alwaysApply rules（自动） |
| P1 | `STATUS.md` + 现状说明书（若有） |
| P1 | 本约束表 / 宪章（复杂任务、改规则、归档时） |
| P2 | 专题 doc、当日 sessions |
| P3 | 早期需求 / guidelines（可能过时） |

推荐开场：`请先读 @docs/STATUS.md + @AGENTS.md，继续 [任务]。`

## 8 · 收工 ritual

| 步 | 做什么 |
|----|--------|
| 1 | 更新 STATUS（`revised_at` 到秒）+「最近决策」 |
| 2 | 必要时改现状说明书 / 专题 YAML / 修订记录 |
| 3 | sessions 写当日 delta（可选） |
| 4 | commit（维护者要求时再 push） |

**禁止**：子任务新建零散 `.cursor/plans/*.plan.md` 代替 STATUS/sessions（除非团队另定）。

## 9 · 绝对禁止（警示）

| # | 禁止 |
|---|------|
| 1 | 脑内模型不完整仍大范围写码 |
| 2 | 用摘要冒充维护者原文（对话归档） |
| 3 | 新决策只留在对话、权威 doc 不更新 |
| 4 | 为精简而删改协作核心章节 |
| 5 | 宪章/约束表写满单一项目路径与事故长文（细则进专题） |
| 6 | 多源任务不写优先级表 |
| 7 | 只改代码、对话新需求不同步文档 |

## 10 · 拷到新项目的最小清单

| 文件 | 作用 |
|------|------|
| 本表 → `docs/AI-COLLABORATION-CONSTRAINTS.md` | 通用约束 |
| `AGENTS.md`（可用 project-context 模板） | 入口 + 索引一行指向本表 |
| `.cursor/rules/project-gotchas.mdc` | alwaysApply：先读 STATUS；复杂任务；多源计划 |
| `docs/STATUS.md` | 进度 |
| （可选）整包 `project-context` Skill | bootstrap + 完整模板 |

```bash
node .agents/skills/project-context/scripts/scaffold.mjs --dry-run
node .agents/skills/project-context/scripts/scaffold.mjs --with-governance
```

复制 Skill 文件夹到目标仓库的 `.agents/skills/project-context/` 后，把本表内容贴进该项目的 `docs/`，并把项目专有路径写进 STATUS / 专题，**不要**写回本表。
