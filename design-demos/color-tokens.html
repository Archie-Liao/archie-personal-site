为什么 --primary 不是颜色名？

代码里的英文名是程序员写的用途标签，不是「这种颜色叫什么」。你只要认中文名 + 色块 + 网站上哪一块。

代码名英文原意实际颜色（中文）网站上指什么
--primary主强调色陶土橙链接、按钮、时间线 Day、图谱节点
--foreground前景字色墨字正文、标题
--background背景奶油纸底整页大底
--punch编辑点缀色焦赭顶栏「日记」、金句竖线、大编号
--card卡片面浅米卡片底卡片、输入框
--accent辅助装饰面暖点缀面金句条、页脚（常为配方混合）
--secondary次要背景面浅米色面金句条渐变配方之一
--primary-deepprimary 加深深陶土链接 hover、金句暗部
--muted-foreground弱化字色弱化灰字日期、说明
--ink-green墨绿（少数真颜色名）墨绿标签

三层模型（L1 / L2 / L3）

  • L1 · 颜料罐：theme.css 里一行 hex（如陶土橙 #CC785C）。改一行 → 全站凡是用这罐色的地方一起变。
  • L2 · 调好的漆：组件里用 color-mix 把几罐 L1 按百分比混合。小样没有单独一块对应该色——例如金句条背景 = 浅米色 + 焦赭 16% + …
  • L3 · 特效漆：某一块复杂渐变，如金句金属字 8 段渐变 → 只改 DailyPunch.tsx。

L1 和 L2 的区别：L1 是「一罐纯色」；L2 是「线上看到的很多实际颜色」，由多罐 L1 调出来。改链接橙 = 改 L1；只改金句条背景更橙、链接不动 = 改 L2 配方里的百分比。

改色最短路径

链接橙再淡一点 → 打开 src/styles/theme.css，搜「陶土橙」或 --primary，改 #CC785C → npm run dev 刷新。

「日记」红再沉一点 → 搜「焦赭」或 --punch，改 #D4562A。

金句字金属感 → 不是改一罐色，改 DailyPunch.tsx 里渐变配方（L3）。

金句条背景更橙、链接别动 → 改 DailyPunch.tsx 里 punch 混合比例（L2）。

完整路由对照、决策树 → docs/COLOR-TOKEN-MAP.md

怎么调（三层模型 · 简表）

  1. L1 Token = 颜料罐标签（theme.css 一行 hex）→ 改罐全站联动
  2. L2 配方 = 几罐按 % 调好的漆（color-mix）→ 只影响那一块 UI
  3. L3 特例 = 金句金属渐变等特效漆 → 见 DailyPunch.tsx
流程：在下面找 token → 查「站点对照」表 → 改 hex → npm run dev 看线上；小样与本站读同一份 theme。

硬约束（试吃笔记）

  • 焦赭点缀 禁止大面积铺底（金句条背景混合 < 30%）
  • 数据红 / 数据灰 仅图表与数据编码
  • 金属字高光：纸色 + 深陶土 / 焦赭，避免纯银灰
